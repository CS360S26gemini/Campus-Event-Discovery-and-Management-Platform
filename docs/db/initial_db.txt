================================================================
  FIRESTORE DATABASE DESIGN
  Campus Event Discovery and Management Platform
  contributor: Yahya
================================================================

NOTE: Firestore is a NoSQL document database. There are no tables,
primary keys, or foreign keys like in a relational DB. Instead,
data is stored in COLLECTIONS of DOCUMENTS, each with fields.
Documents can also have SUBCOLLECTIONS nested inside them.
References between documents are just stored as plain string IDs.


================================================================
  SECTION 1 -- FULL DATABASE TREE
================================================================

FIRESTORE ROOT
|
+-- users/
|   +-- {userId}/                        (document per user)
|       +-- [fields]
|       +-- saved_events/                (subcollection)
|       |   +-- {eventId}
|       +-- rsvps/                       (subcollection)
|       |   +-- {eventId}
|       +-- memories/                    (subcollection)
|           +-- {memoryId}
|
+-- events/
|   +-- {eventId}/                       (document per event)
|       +-- [fields]
|       +-- ratings/                     (subcollection)
|       |   +-- {userId}
|       +-- attendees/                   (subcollection)
|           +-- {userId}
|
+-- event_proposals/
|   +-- {proposalId}/                    (document per submission)
|       +-- [fields]
|
+-- reports/
|   +-- {reportId}/                      (document per SOS report)
|       +-- [fields]
|
+-- notifications/
|   +-- {userId}/
|       +-- messages/                    (subcollection)
|           +-- {notificationId}
|
+-- app_config/
    +-- settings                         (single document)


================================================================
  SECTION 2 -- COLLECTIONS IN DETAIL
================================================================

----------------------------------------------------------------
  users / {userId}
  (document ID = Firebase Auth UID, auto-assigned on signup)
----------------------------------------------------------------

  fullName             string
  email                string
  role                 string       "attendee" / "organizer" / "admin"
  university           string       e.g. "LUMS"
  location             string       e.g. "Lahore"
  profilePicUrl        string       Firebase Storage URL
  interests            array        e.g. ["Music", "Sports", "Career"]
  darkMode             boolean      true = dark theme (default)
  fcmToken             string       for push notifications

  googleCalendarToken  map
    |-- accessToken    string
    |-- refreshToken   string
    +-- expiresAt      timestamp

  createdAt            timestamp

  -- Subcollection: saved_events / {eventId} --
    eventId            string
    title              string       (copied here for fast display)
    date               timestamp
    category           string
    thumbnailUrl       string
    savedAt            timestamp

  -- Subcollection: rsvps / {eventId} --
    eventId            string
    title              string       (copied here for fast display)
    date               timestamp
    status             string       "confirmed" / "cancelled" / "attended"
    checkedIn          boolean
    qrCodeToken        string       unique token encoded in the QR code
    addedToCalendar    boolean
    gcalEventId        string       Google Calendar event ID (for un-RSVP removal)
    rsvpAt             timestamp

  -- Subcollection: memories / {memoryId} --
    eventId            string
    eventTitle         string
    photoUrls          array        Firebase Storage URLs
    attendedAt         timestamp
    rating             number       1-5 stars given by user


----------------------------------------------------------------
  events / {eventId}
  (document ID = auto-generated by Firestore)
----------------------------------------------------------------

  title                string       e.g. "Hassan Raheem Concert"
  description          string
  category             string       "Performance" / "Career" / "Sports"
                                    "Academic" / "Arts" / "Talks"
  tags                 array        e.g. ["Music", "Live", "Concert"]
                                    matched against user interests
  date                 timestamp
  endTime              timestamp
  location             string       e.g. "LUMS Campus"
  capacity             number       e.g. 1000
  rsvpCount            number       auto-updated (shows "0 / 1000")
  checkedInCount       number       auto-updated on QR scan
  thumbnailUrl         string       Firebase Storage URL
  trailerUrl           string       Firebase Storage URL (optional video)
  sponsors             array        e.g. ["University Partners"]
  foodStalls           array        e.g. ["Food & Merch"]
  organizerId          string       references users/{userId}
  organizerName        string       (copied for fast display)
  isVerified           boolean      admin sets this, shows checkmark badge
  averageRating        number       auto-calculated from ratings subcollection
  ratingCount          number
  status               string       "active" / "cancelled" / "completed"
  createdAt            timestamp

  -- Subcollection: ratings / {userId} --
    userId             string
    stars              number       1-5
    review             string       optional text
    ratedAt            timestamp

  -- Subcollection: attendees / {userId} --
    userId             string
    fullName           string       (copied for organizer view)
    qrToken            string       must match users/{uid}/rsvps/{eid}.qrCodeToken
    checkedIn          boolean
    checkedInAt        timestamp


----------------------------------------------------------------
  event_proposals / {proposalId}
  (same fields as an event, plus the ones below)
----------------------------------------------------------------

  title                string
  description          string
  category             string
  tags                 array
  date                 timestamp
  location             string
  capacity             number
  sponsors             array
  foodStalls           array
  trailerUrl           string
  organizerId          string
  organizerName        string
  status               string       "pending" / "approved" / "rejected"
  adminNote            string       reason if rejected
  submittedAt          timestamp
  reviewedAt           timestamp

  FLOW: Organizer fills Propose Event form
     -> document created here with status "pending"
     -> Admin reviews and approves or rejects
     -> If approved, Cloud Function copies it into events/ collection
     -> Organizer receives push notification of the decision


----------------------------------------------------------------
  reports / {reportId}
  (SOS button and harassment reports)
----------------------------------------------------------------

  reporterId           string       null if anonymous
  reporterName         string       null if anonymous
  isAnonymous          boolean
  description          string
  location             map
    |-- lat            number
    +-- lng            number
  status               string       "open" / "reviewed" / "resolved"
  adminNote            string
  submittedAt          timestamp
  resolvedAt           timestamp


----------------------------------------------------------------
  notifications / {userId} / messages / {notificationId}
----------------------------------------------------------------

  title                string       e.g. "Your event was approved!"
  body                 string       e.g. "Hassan Raheem Concert is now live"
  type                 string       "event_approved" / "event_rejected"
                                    "rsvp_reminder" / "new_event" / "countdown"
  eventId              string       optional link to related event
  isRead               boolean
  createdAt            timestamp


----------------------------------------------------------------
  app_config / settings
  (single document, admin-controlled)
----------------------------------------------------------------

  maintenanceMode      boolean
  featuredEventIds     array        hand-picked events shown on home screen


================================================================
  SECTION 3 -- FEATURE TO DATABASE MAPPING
================================================================

  FEATURE                          WHERE IT LIVES
  -------                          --------------
  User login / signup              Firebase Auth + users/{userId}
  Edit profile                     users/{userId} fields
  Dark mode toggle                 users/{userId}.darkMode
  Interest selection (onboarding)  users/{userId}.interests
  "For You" suggestions            events.tags matched against users.interests
  Upcoming events (home screen)    events/ where status=active, ordered by date
  Ticket availability (0/1000)     events.rsvpCount vs events.capacity
  Favourite / save event           users/{userId}/saved_events
  RSVP / Secure Your Spot          users/{userId}/rsvps + events.rsvpCount
  QR code check-in                 rsvps.qrCodeToken verified in events/attendees
  Add to Google Calendar           users.googleCalendarToken + rsvps.gcalEventId
  Countdown after RSVP             rsvps.date computed client-side
  Event verification badge         events.isVerified set by admin
  Organizer propose event          event_proposals collection
  Admin approve / reject event     event_proposals.status + Cloud Function
  Manage attendees (organizer)     events/{eventId}/attendees
  Sponsors and food stalls         events.sponsors + events.foodStalls
  Event video trailer              events.trailerUrl via Firebase Storage
  Rating system                    events/{eventId}/ratings subcollection
  SOS / harassment report          reports collection
  Push notifications               notifications/{userId}/messages via FCM
  Post-event memories / photos     users/{userId}/memories + Firebase Storage


================================================================
  SECTION 4 -- FIREBASE STORAGE FOLDERS
  (separate from Firestore, stores actual files)
================================================================

  profile_pictures/{userId}        user profile photos
  event_thumbnails/{eventId}       event banner images
  event_trailers/{eventId}         video trailers (max 50MB)
  memories/{userId}/{eventId}      user post-event photos
